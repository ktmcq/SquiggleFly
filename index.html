SquiggleFly

About


SquiggleFly allows users to fly drones on complicated or specific paths without having to worry about coding them or doing so manually.

SquiggleFly makes flying drones accessible to everyone.

SquiggleFly was created by Sean Crotty, Jack Masciopinto, Katie McHugh, and Jack Morgan. It was developed through an Agile process over the course of 4 weeks.

Requirements


  • When the user draws a pattern on their phone, that same pattern should be sent to the drone.
  • The points which make up the pattern must be collected as the pattern is being drawn.
  • The drone will hover at 10 meters until it receives instructions from the application regarding the path it should fly.
  • Once the drone receives a pattern from the application, it will immediately start executing that fly pattern.
  • When the drone finishes flying the path it was sent, it will continue to hover until it receives further instructions from the application.
  • There must be some way for the users to exit the program with a failsafe.
  • The user should be able to view the actual path that was flown by the drone layered against the path that they drew.
  • We had to make sure that the canvas was of specified dimensions so that the user could not fly the drone beyond a certain distance away from its starting point.
  • We had to make sure that the orientation of the canvas matched the way in which the drone actually flies so there are no surprises.

Installation


root@my-machine:~$ git clone https://github.com/SAREC-Lab/SquiggleFly.git
root@my-machine:~$ cd SquiggleFly
root@my-machine:~/SquiggleFly$ pip install -r requirements

Usage


Terminal 1

root@my-machine:~/SquiggleFly$ python drone_controller.py

Terminal 2

root@my-machine:~$ firefox

  1. Once your drone is ready to connect to your laptop, run the commands above in two separate terminals as shown.
  2. In the Firebox browser which pops up, navigate to the index.html file within the SquiggleFly repo.
  3. You will see a view like the one below. This is where you will draw the path you would like for your drone to fly. You can clear the path at any point to start over by clicking "Clear Path". When you would like your drone to fly the path, just click "Fly Path", and SquiggleFly will do the rest. You can check the progress of your drone in Terminal 1.

Architecture


Our project is a web application which communicates with a drone via Mavlink. We used Flask as our web framework, as it is written in Python, and we are most comfortable using Python to communicate with and control the drone. We integrated the Python code for the drones into the Python file which runs the web application. We implemented a drawing canvas feature through JavaScript on the frontend, and our coordinate-gathering algorithm on the frontend smartly turns the path drawn into a list of coordinates. Once the "Fly Path" button has been clicked after a path has been drawn, those points are received on the backend and transformed into real-world coordinates based on the current position of the drone, and the drone then flies the path based on those waypoints.
